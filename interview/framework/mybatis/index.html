    – MyBatis #

    • MyBatis

      是一个基于java的持久层框架
它内部封装了JDBC，使开发者只需要专注于sql语句本身

    • 特点

      1. 与JDBC相比，减少了50%以上的代码量
2. 基于SQL语句编程，相当灵活。
3. SQL写在XML里，降低了耦合度，便于统一管理和优化，且可重用
4. 提供XML标签，支持编写动态SQL语句
5. 提供映射标签，支持对象与数据库的ORM字段关系映射
6. 兼容各种数据库，能够与Spring很好的集成

    – MyBatis 缓存机制 # 

    1. mybatis提供查询缓存，用于减轻数据压力，提高数据库性能
2. 缓存失效：查询条件不相同，或两次查询之间出现了增删改操作
    缓存 级别 开启 缓存范围
    一级缓存 SqlSession 级别 默认开启 同一个SqlSession，即数据库会话
    二级缓存 Mapper 级别 配置开启 同一个Mapper，即同一个 namespace 的 mapper.xml

    – JDBC 编程步骤 #

    • 加载驱动

      Class.forName("com.mysql.jdbc.Driver");

    • 建立连接

      String user = "xxx";
String password = "xxx";
String url = "jdbc:mysql://127.0.0.1:3306/xxx?useUnicode=true";
Connection conn = DriverManager.getConnection(url, user, password);

    • 创建 Statement

      String sql ="SELECT * FROM catalog_info WHERE catalog_name=?";
PreparedStatement prs = conn.prepareStatement(sql);
prs.setString(1, "JavaSE");

    • 执行sql语句，获取结果集

      ResultSet res = prs.executeQuery();
while (res.next()) {
    System.out.println(res.getLong("catalog_id") + " - " + res.getString("catalog_name"));
}

    • 关闭资源

      res.close();
prs.close();
conn.close();

    – #{}、${} #

    含义 Mybatis 处理过程 防止SQL注入
    #{} 预编译处理 将 #{} 替换为 ? 号，
    再调用PreparedStatement的set方法来赋值    		 可以
    ${} 字符串替换 将 ${} 替换成变量的值 不能
